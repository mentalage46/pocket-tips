# Prompting Tips

## 핵심 원칙

- **명확하게** - 모호한 표현 피하기
- **구체적으로** - 원하는 형식, 길이, 스타일 명시
- **역할 부여** - "너는 ~전문가야" 로 시작
- **예시 제공** - 원하는 출력 형태 보여주기

## 효과적인 기법

| 기법             | 설명             | 예시                      |
| ---------------- | ---------------- | ------------------------- |
| Few-shot         | 예시 2-3개 제공  | `예: A→B, C→D, E→?`       |
| Chain of Thought | 단계별 사고 유도 | `"단계별로 생각해봐"`     |
| 제약 조건        | 범위 제한        | `"3문장으로"`, `"코드만"` |
| 페르소나         | 역할 지정        | `"시니어 개발자로서"`     |

## 구조화 팁

```
[역할] 너는 ~이다
[맥락] 상황 설명
[작업] 해야 할 일
[형식] 출력 형태
[제약] 조건/제한
```

## 자주 쓰는 표현

- `"~처럼 행동해"` - 페르소나 설정
- `"단계별로"` - 복잡한 문제 분해
- `"요약해줘"` / `"자세히 설명해"` - 길이 조절
- `"~형식으로"` - 출력 포맷 지정
- `"왜 그런지 설명해"` - 추론 과정 요청

## 피해야 할 것

- ❌ 너무 긴 프롬프트 (핵심만)
- ❌ 여러 질문 한번에 (하나씩)
- ❌ 모호한 표현 ("좋게 해줘")
- ❌ 맥락 없는 질문

## 📋 복사용 템플릿

### 기본 템플릿

```
너는 [분야] 전문가야.
[배경/상황 설명]
[구체적인 요청]
[출력 형식] 형태로 답변해줘.
```

### 코드 리뷰 요청

```
시니어 개발자로서 아래 코드를 리뷰해줘.
- 버그 가능성
- 성능 개선점
- 가독성 제안
각 항목을 bullet point로 정리해줘.

[코드]
```

### 문서/글 작성

```
[주제]에 대해 [대상 독자]를 위한 글을 써줘.
- 분량: [N]자 내외
- 톤: [친근한/전문적인/격식있는]
- 포함할 내용: [핵심 포인트]
```

### 문제 해결/디버깅

```
아래 문제를 단계별로 분석해줘.
1. 문제 원인 파악
2. 가능한 해결책 3가지
3. 추천 해결책과 이유

[문제 상황]
```

### 요약 요청

```
아래 내용을 [N]문장으로 요약해줘.
핵심 포인트만 bullet point로 정리해줘.

[내용]
```

### 번역/교정

```
아래 텍스트를 [언어]로 번역해줘.
- 자연스러운 표현 사용
- 원문 뉘앙스 유지

[텍스트]
```

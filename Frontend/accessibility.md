# 웹 접근성 (Web Accessibility) 가이드라인

웹 접근성은 장애인, 고령자 등 모든 사용자가 웹 사이트에서 제공하는 정보를 동등하게 접근하고 이해할 수 있도록 보장하는 것을 의미합니다.

## 1. 구현 방법

### 1.1. 시맨틱 HTML (Semantic HTML)

가장 기본적이고 강력한 접근성 구현 방법은 의미에 맞는 HTML 태그를 사용하는 것입니다.

- **`<header>`, `<nav>`, `<main>`, `<footer>`**: 페이지 구조를 명확히 정의합니다.
- **`<h1>` ~ `<h6>`**: 콘텐츠의 계층 구조를 나타냅니다.
- **`<button>` vs `<a>`**: 동작을 수행할 때는 `button`, 페이지 이동 시에는 `a` 태그를 사용합니다.
- **`<label>`**: 폼 요소(`input`, `select` 등)에 반드시 연결하여 스크린 리더가 용도를 알 수 있게 합니다.

### 1.2. WAI-ARIA (Web Accessibility Initiative - Accessible Rich Internet Applications)

HTML만으로 표현하기 어려운 동적인 UI나 복잡한 위젯의 접근성을 향상시키기 위해 사용합니다.

- **Role (역할)**: 요소의 목적을 정의합니다. (예: `role="tablist"`, `role="dialog"`)
- **State (상태)**: 요소의 현재 상태를 나타냅니다. (예: `aria-expanded="true"`, `aria-selected="false"`)
- **Property (속성)**: 요소의 특징이나 관계를 정의합니다. (예: `aria-label="닫기"`, `aria-labelledby="ID"`)

> [!IMPORTANT] > **ARIA의 제1원칙**: HTML5 태그로 구현 가능하다면 ARIA를 사용하지 마세요. ARIA는 보조적인 수단입니다.

## 2. 주요 고려사항

### 2.1. 시각적 접근성

- **색상 대비**: 텍스트와 배경의 명도 대비를 최소 4.5:1 이상으로 유지합니다.
- **대체 텍스트**: 모든 이미지(`<img>`)에는 `alt` 속성을 제공합니다. 의미 없는 장식용 이미지는 `alt=""`로 비워둡니다.
- **확대/축소**: 브라우저에서 200% 확대하더라도 콘텐츠가 겹치거나 잘리지 않아야 합니다.

### 2.2. 키보드 접근성

- **초점(Focus) 표시**: 현재 어디에 초점이 있는지 시각적으로 명확히 표시해야 합니다. (`outline: none` 지양)
- **논리적 탭 순서**: Tab 키 이동 순서가 시각적인 흐름과 일치해야 합니다.
- **키보드 트랩 방지**: 키보드만으로 모든 기능을 사용할 수 있어야 하며, 특정 요소에 갇히지 않아야 합니다.

### 2.3. 청각 및 인지적 접근성

- **자막 및 스크립트**: 영상 콘텐츠에는 자막을, 오디오에는 대본을 제공합니다.
- **일관된 내비게이션**: 사이트 전체에서 일관된 메뉴 구조와 명칭을 사용합니다.
- **충분한 시간 제공**: 세션 만료나 입력 제한 시간이 있는 경우, 시간을 연장할 수 있는 수단을 제공합니다.

## 3. 웹 접근성 체크리스트

| 구분              | 체크 항목                                               | 확인 여부 |
| :---------------- | :------------------------------------------------------ | :-------: |
| **인식의 용이성** | 모든 비텍스트 콘텐츠에 대체 텍스트를 제공했는가?        |    [ ]    |
|                   | 색상만으로 정보를 구분하게 하지 않았는가?               |    [ ]    |
|                   | 텍스트와 배경의 대비가 충분한가? (4.5:1 이상)           |    [ ]    |
| **운용의 용이성** | 모든 기능을 키보드로 사용할 수 있는가?                  |    [ ]    |
|                   | 초점(Focus) 이동 순서가 논리적인가?                     |    [ ]    |
|                   | 페이지, 프레임, 콘텐츠 블록에 적절한 제목을 제공했는가? |    [ ]    |
| **이해의 용이성** | 기본 언어를 명시했는가? (`<html lang="ko">`)            |    [ ]    |
|                   | 입력 오류 시 사용자에게 명확하게 안내하는가?            |    [ ]    |
|                   | 레이블(`label`)이 폼 컨트롤과 적절하게 연결되었는가?    |    [ ]    |
| **견고성**        | 마크업 문법을 준수했는가? (태그 열고 닫기, 중복 ID 등)  |    [ ]    |
|                   | 웹 표준 및 ARIA를 올바르게 사용했는가?                  |    [ ]    |
